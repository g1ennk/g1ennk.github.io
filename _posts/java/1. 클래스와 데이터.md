## 클래스와 데이터

### 클래스가 필요한 이유

- 클래스가 없다면 너무나도 많은 변수를 핸들링 해야한다.
- 하지만 클래스를 통해 해당 클래스 안에 정의한 멤버 변수 혹은 필드를 통해 문제를 해결할 수 있다.
- 관례적으로 클래스 이름은 대문자로 시작하고 낙타 표기법을 사용한다. ex.UserService
- 설계도 = 클래스, 설계도를 통해 만들어진 실체 = 객체 혹은 인스턴스

```java
Student student = new Student(); // 1. student이라는 변수 이름을 가진 Student 객체 생성
Student student = x001; // 2.new 키워드로 x001이라는 참조값을 student이라는 변수에 보관
```

### 객체 사용하기

- 객체에 접근하려면 `.`(점,dot)를 사용하면 된다. ex) student.age == x001.age

### 정리

1. Class: 객체를 생성하기 위한 설계도로 필드와 메서드를 정의할 수 있다.
2. Object: 클래스에서 정의한 속성과 기능을 가진 실체이다. ex) 자동차 설계도를 통해 만들어진 실제 자동차
3. Instance: 특정 클래스를 통해 생성된 객체를 의미한다.
4. Object vs Instance: 인스턴스는 객체보다 조금 더 관계에 초점을 맞춘 단어이지만, 보통 둘을 구분하지 않고 사용한다.
